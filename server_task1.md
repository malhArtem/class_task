## Задание: «Исследователь файловой системы и управление процессами»

Вы зашли на сервер (Ubuntu/Debian/CentOS) под обычным пользователем. У вас нет прав root, но есть доступ к консоли.

### Часть 1. Навигация и файлы (5 простых шагов)

Выполните последовательно:

1. **Оказавшись в домашней директории**, создайте папку с вашей фамлией, а затем создайте внутри неё три вложенные папки:  
   `project/docs`, `project/backups`, `temp`

2. **Внутри `project/docs`** создайте три файла:  
   - `readme.txt` со строкой `"Version 1.0"`  
   - `notes.txt` — пустой  
   - `data.txt` со строками:  
     ```
     apple
     banana
     cherry
     date
     ```

3. **Скопируйте** `data.txt` в папку `project/backups` с именем `data_backup.txt`

4. **Переместите** пустой `notes.txt` из `project/docs` в `temp`

5. **Удалите** папку `temp` вместе со всем её содержимым (да, она всё ещё пустая — но команда должна работать и для непустой)

---

### Часть 2. Поиск и фильтрация текста

Не выходя из домашней директории:

6. **Найдите** все файлы с расширением `.txt` внутри папки `project` (включая подпапки)

7. **Подсчитайте**, сколько строк в файле `project/docs/data.txt`

8. **Выведите на экран** только строки из `project/docs/data.txt`, которые содержат букву `a`

9. **Замените** слово `banana` на `blueberry` в файле `project/docs/data.txt` (без открытия редактора — одной командой)

10. **Проверьте**, что замена произошла: выведите содержимое файла на экран

---

### Часть 3. Процессы и система (всё безопасно, ничего не ломаем)

11. **Выведите** список всех процессов, запущенных от вашего пользователя

12. **Найдите** PID процесса `sshd` (если его нет — `systemd` или `bash`)

13. **Выведите** в удобном для человека виде, сколько памяти свободно на сервере

14. **Узнайте**, какая у вас операционная система и версия ядра (одной командой)

15. **Выполните** команду `yes > /dev/null &` — она запустит бесконечный процесс в фоне.  
    - Найдите его PID  
    - Остановите процесс (командой `kill`)

---

### Часть 4. Проверка (что сдать преподавателю)

Студент должен **показать в консоли** или прислать лог сессии, где видны:

- Результат `ls -R project` после шага 5
- Результат шага 8 (строки с `a`)
- Результат шага 10 (файл с `blueberry`)
- Команды шагов 13–15 с их выводом

---

### Подсказки для начинающих (раздатка)

| Что сделать | Команда |
|------------|---------|
| Создать папку | `mkdir -p папка/вложенная` |
| Создать файл с текстом | `echo "текст" > файл.txt` |
| Посмотреть файл | `cat файл.txt` |
| Скопировать | `cp исходник куда/` |
| Переместить/переименовать | `mv что куда` |
| Удалить папку с содержимым | `rm -rf папка` |
| Найти файлы | `find project -name "*.txt"` |
| Подсчитать строки | `wc -l файл` |
| Отфильтровать строки | `grep "буква" файл` |
| Заменить в файле | `sed -i 's/старое/новое/g' файл` |
| Список процессов | `ps aux` |
| Найти процесс по имени | `pgrep имя` |
| Свободная память | `free -h` |
| ОС и ядро | `uname -a` или `cat /etc/os-release` |
| Убить процесс | `kill PID` |
