# Лабораторная работа № 3.2
## Конфигурирование локальной безопасности в Windows (на основе Windows 7 с проекцией на Windows 10)

**Дисциплина:** Конфигурирование Windows  
**Модуль:** 3.2. Конфигурирование безопасности в Windows 10  
**Время выполнения:** 2 академических часа (90 минут)  
**Оборудование:** Виртуальная машина с ОС Windows 7 (Professional / Enterprise / Ultimate). Учетная запись — администратор.

---

## Цель работы

Изучить оснастку «Локальные политики безопасности» (secpol.msc), научиться настраивать политики паролей, аудит событий и параметры контроля учетных записей (UAC) в Windows 7. Полученные навыки перенести на понимание безопасности Windows 10, указав различия в механизмах защиты.

> **Важно:** Современные средства безопасности Windows 10 (Device Guard, Credential Guard, динамическая блокировка, изолированный браузер) отсутствуют в Windows 7. Однако базовые политики аутентификации, разграничения доступа и аудита остаются общими, что и позволяет выполнять данную работу на имеющемся оборудовании.

---

## Теоретическое введение (обязательно к прочтению)

Безопасность операционной системы строится на нескольких уровнях:

1. **Аутентификация** — проверка подлинности пользователя (пароли, PIN, биометрия).
2. **Авторизация** — определение прав доступа к объектам (файлам, папкам, реестру).
3. **Аудит** — регистрация событий безопасности для последующего анализа.
4. **Контроль учетных записей (UAC)** — предотвращение несанкционированного изменения системных параметров.

Windows 10 добавила следующие важные элементы, которых нет в Windows 7:
- **Windows Defender Antivirus** (полноценный антивирус, а не только антишпион);
- **Device Guard / Credential Guard** (защита хешей паролей в памяти ядра, требует виртуализации);
- **Dynamic Lock** (автоматическая блокировка ПК при удалении сопряженного телефона по Bluetooth);
- **SmartScreen** (проверка репутации файлов и сайтов на уровне ОС).

Однако **локальные политики безопасности** (secpol.msc), с которыми вы будете работать сегодня, практически идентичны в Windows 7, 8.1 и 10. Это означает, что, научившись настраивать безопасность на Windows 7, вы без труда сделаете это и на Windows 10.

## Ход выполнения работы

### 1. Подготовка к работе (5 минут)

1. Запустите виртуальную машину с Windows 7.
2. Войдите в систему под именем `Administrator` (пароль, если установлен — введите; если нет — задайте его сейчас: `P@ssw0rd`).
3. Убедитесь, что сетевая карта настроена на режим «Внутренняя сеть» или «Только хост» (чтобы исключить внешние атаки).

**Действие для отчета:**  
> Сделайте скриншот рабочего стола Windows 7 с открытым окном `Пуск → Компьютер → Свойства системы` (чтобы было видно версию ОС и имя компьютера). Подпишите скриншот: *«Рисунок 1 — Исходное состояние ВМ Windows 7»*.

---

### 2. Знакомство с оснасткой «Локальные политики безопасности»

Нажмите `Пуск → Выполнить` (или `Win + R`), введите команду:

```cmd
secpol.msc
```

Откроется окно «Локальные политики безопасности». Обратите внимание на древовидную структуру слева. Основные разделы:

- **Политики учетных записей** — управляют паролями и блокировкой пользователей.
- **Локальные политики** — включают «Назначение прав пользователя» (кому что разрешено) и «Параметры безопасности» (различные флаги защиты).
- **Политики аудита** — определяют, какие события записывать в журнал безопасности.

**Скриншот для отчета:**  
> Откройте secpol.msc полностью, разверните окно на весь экран. Сделайте скриншот, подписав: *«Рисунок 2 — Оснастка локальных политик безопасности»*.

---

### 3. Настройка политики паролей (имитация требований Windows 10)

В Windows 10 корпоративные стандарты часто требуют, чтобы пароль был сложным, длиной не менее 8 символов и не совпадал с предыдущими 5 паролями. Настроим то же самое в Windows 7.

**Действия:**

1. В левой панели secpol.msc выберите: `Политики учетных записей → Политика паролей`.
2. Дважды щелкните на каждом параметре и установите значения:

| Параметр | Значение |
|----------|----------|
| Пароль должен отвечать требованиям сложности | **Включен** |
| Минимальная длина пароля | **8 символов** |
| Хранить пароли для обратимого шифрования | **Отключен** (никогда не включайте) |
| Пароль должен храниться с использованием обратимого шифрования | **Отключен** |
| Срок действия пароля (не трогаем) | 42 дня (оставить по умолчанию) |

3. Теперь перейдите в раздел `Политики учетных записей → Политика блокировки учетной записи`. Установите:
   - **Порог блокировки** = `5` неверных попыток.
   - **Сброс счетчика блокировки через** = `30` минут.
   - **Длительность блокировки учетной записи** = `30` минут.

> **Пояснение:** Если кто-то 5 раз подряд введет неверный пароль, учётная запись будет заблокирована на полчаса. Это защита от подбора пароля (брутфорс). Windows 10 по умолчанию использует аналогичный механизм.

**Скриншот для отчета:**  
> Сделайте скриншот окна с открытым разделом «Политика паролей» и «Политика блокировки», где видны установленные вами значения. Подпись: *«Рисунок 3 — Настроенные политики паролей и блокировки»*.

---

### 4. Управление правами пользователей и усиление ограничений

В Windows 10 для повышения безопасности часто запрещают интерактивный вход для встроенной учетной записи `Guest`, а также ограничивают, кто может запускать службы.

**Выполните следующие шаги:**

1. В secpol.msc откройте: `Локальные политики → Назначение прав пользователя`.
2. Найдите пункт **«Отказ во входе через сеть»**. Дважды щелкните по нему. Нажмите «Добавить пользователя или группу» → введите `Guest` → ОК.
3. Найдите пункт **«Работа в качестве службы»**. Удалите оттуда всех, кроме `LOCAL SERVICE` и `NETWORK SERVICE` (выделите лишнего и нажмите «Удалить»).  
   > *Зачем это?* Если злоумышленник получит права обычного пользователя, он не сможет внедрить вредоносную службу, поскольку для запуска служб нужны специальные права.

4. Найдите пункт **«Доступ к этому компьютеру из сети»**. Убедитесь, что там только `Administrators` и `Users`. Если есть `Everyone` — удалите.

**Скриншот для отчета:**  
> Откройте окно «Работа в качестве службы» так, чтобы был виден список учетных записей. Сделайте скриншот. Подпись: *«Рисунок 4 — Права на запуск служб ограничены»*.

---

### 5. Настройка аудита событий безопасности

Аудит позволяет отслеживать, кто и когда пытался войти в систему, менял политики или получал доступ к файлам. В Windows 10 аудит является основой для расследования инцидентов.

**Действия:**

1. В secpol.msc перейдите: `Локальные политики → Политики аудита`.
2. Для каждого из следующих пунктов установите флажки **«Успех»** и **«Отказ»** (оба):
   - Аудит входа в систему
   - Аудит доступа к объектам
   - Аудит изменения политики
   - Аудит управления учетными записями
   - Аудит отслеживания процессов (по желанию)

**Важно:** После настройки аудита необходимо перезапустить службу «Безопасность» или перезагрузить компьютер. Проще всего выполнить перезагрузку.

> **Скриншот:** Сделайте скриншот окна «Политики аудита» с включенными галочками. Подпись: *«Рисунок 5 — Включенный аудит системы»*.

---

### 6. Практическая проверка аудита: генерация события неудачного входа

Теперь проверим, действительно ли события пишутся в журнал «Безопасность».

**Пошаговая инструкция:**

1. Создайте тестового пользователя:
   - Откройте `Пуск → Панель управления → Администрирование → Управление компьютером`.
   - Разверните «Локальные пользователи» → «Пользователи».
   - На пустом месте нажмите правой кнопкой → «Новый пользователь».
   - Имя: `TestUser`, пароль: `P@ssw0rdTest`. Снимите галочку «Требовать смену пароля при следующем входе».
   - Нажмите «Создать», затем «Закрыть».

2. **Выйдите из системы** (`Пуск → Завершение работы → Сменить пользователя`).

3. На экране приветствия выберите `TestUser`. **Намеренно введите неправильный пароль** два или три раза (например, `111`). Затем нажмите «ОК».

4. После второго-третьего неправильного ввода снова войдите как `Administrator`.

5. Откройте `Пуск → Выполнить → eventvwr.msc` (Просмотр событий).
   - Слева выберите: `Журналы Windows → Безопасность`.
   - В средней панели нажмите «Фильтр текущего журнала...» и укажите **ID события: 4625** (неудачная попытка входа).
   - Вы увидите одну или несколько записей.

6. Щелкните по любой записи 4625. В нижней области будет описание:  
   *«Сбой при входе в систему»* → посмотрите поле **«Имя учетной записи»** — там должно быть `TestUser`, а поле **«Причина»** — *«неизвестное имя пользователя или неверный пароль»*.

**Скриншот для отчета:**  
> Сделайте скриншот окна eventvwr.msc с отфильтрованными событиями 4625, где видно имя TestUser и причину отказа. Подпись: *«Рисунок 6 — Журнал безопасности фиксирует попытки взлома»*.

### 7. Работа с UAC (Контроль учетных записей)

UAC — это механизм, который запрашивает подтверждение (или пароль администратора) при попытке выполнить действия, влияющие на систему.

**Выполните:**

1. Зайдите под `TestUser` (не администратор).
2. Откройте `Панель управления → Учетные записи пользователей → Изменение параметров контроля учетных записей`.
   - Переместите ползунок на самый верх (Всегда уведомлять).
   - Нажмите «ОК».
3. Попробуйте **создать текстовый файл** в папке `C:\Windows`:
   - Правой кнопкой на рабочем столе → Создать → Текстовый документ → назовите `test.txt`.
   - Попытайтесь перетащить его в `C:\Windows`.
4. Что произошло? Появилось окно UAC с запросом пароля администратора. Введите пароль от `Administrator`, после чего операция разрешится.

> **Пояснение для отчета:** В Windows 10 UAC работает аналогично, но добавлена защита от подделки окон UAC (запросы идут через Secure Desktop). Также в Windows 10 можно настраивать UAC для отдельных приложений через реестр.

**Скриншот для отчета:**  
> Сделайте скриншот именно в тот момент, когда окно UAC запрашивает пароль администратора при попытке скопировать файл в C:\Windows. Подпись: *«Рисунок 7 — Запрос UAC при доступе к защищенной папке»*.

### 8. Дополнительные параметры безопасности (важные для Windows 10)

В Windows 10 по умолчанию отключено анонимное перечисление учетных записей SAM. Это мешает злоумышленнику удаленно узнать список пользователей. Настроим то же самое в Windows 7.

1. В `secpol.msc` перейдите: `Локальные политики → Параметры безопасности`.
2. Найдите: **«Сетевой доступ: не разрешать анонимное перечисление SAM-аккаунтов»** → установите **Включен**.
3. Также найдите: **«Сетевая безопасность: уровень аутентификации LAN Manager»** → установите **«Отправлять только NTLMv2. Отказывать LM и NTLM»** (это отключает устаревшие и небезопасные протоколы).

> **Важно для сдачи:** В Windows 10 эти параметры выставлены автоматически при установке.

**Скриншот:**  
> Откройте указанные параметры, сделайте скриншот окна с их значениями. Подпись: *«Рисунок 8 — Отключение устаревших протоколов и анонимного доступа к SAM»*.
