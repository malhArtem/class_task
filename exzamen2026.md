

Модуль 4. Разработка информационной системы (Учёт заказов клиентов)

### Цель

Разработать приложение для учёта заказов: авторизация, две роли («менеджер» и «кладовщик»), просмотр и изменение заказов.

### 4.1. База данных

#### База данных:

База данных состоит из двух таблиц.

Таблица «Пользователи» хранит учётные записи сотрудников.

* Идентификатор — уникальный номер пользователя.
* Логин — уникальное имя для входа.
* Пароль — хранится в открытом виде.
* Полное имя — ФИО сотрудника.
* Роль — может быть либо «менеджер», либо «кладовщик».

Таблица «Заказы» содержит данные о клиентских заказах.

* Идентификатор заказа — уникальный номер.
* Имя клиента — наименование заказчика.
* Товар — наименование продукции.
* Количество — число единиц (целое положительное).
* Статус — состояние выполнения заказа: «в работе» или «завершён».

#### Тестовые данные:

* Менеджер: login: admin, pass: 123, роль менеджер.
* Кладовщик: login: user, pass: 123, роль кладовщик.
* 3–4 заказа со статусами.

### 4.2. Функционал

#### Общее:

* Форма входа: логин/пароль, проверка, блокировка после 3 неудачных попыток (учётная запись деактивируется).
* При успешном входе открывается таблица заказов.

#### Менеджер:

* Может добавлять новые заказы (все поля).
* Может менять статус заказа (на любой).
* Может удалять заказ.

#### Кладовщик:

* Может посмотреть список заказов.
* Не может добавлять/редактировать/удалять заказы.

#### Интерфейс:

· После входа – таблица заказов.
· Для менеджера – кнопки «Добавить», «Изменить статус», «Удалить».
· Для кладовщика – список заказов.
