# Лабораторная работа №5  
## «Сетевой детектив: диагностика и документирование»

---

### Введение

Вы зашли на сервер. До вас здесь уже кто-то работал. Сервер ведёт себя подозрительно: есть странные файлы, неизвестные процессы, кто-то стучится наружу.

У вас нет документации, нет паролей от других машин, есть только консоль.

**Ваша задача** — обследовать сервер, найти все аномалии, идентифицировать подозрительные процессы и **задокументировать** их. Останавливать процессы **нельзя** — вы работаете на общем сервере с другими студентами.

---

### Правила работы

- **Не останавливайте процессы** (не используйте `kill`)
- **Не удаляйте файлы** в `/tmp` и других общих директориях
- Все команды записывайте в отчёт вместе с выводом
- Работайте аккуратно, не мешайте другим студентам

## Часть 2. Исследование подозрительных процессов

**Ситуация:**  
В директории `/tmp` есть файл с именем `.secret.log`. Он постоянно обновляется (меняется время изменения, добавляются новые строки). Нужно найти, какой процесс в него пишет.

**Что делать:**  
Выясните, какой процесс держит этот файл открытым для записи.

**Что записать:**  
Имя процесса, его PID и первую строку из открытого файла (можно `head -1`).

**Что записать:**

- Рабочую директорию

## Команды-шпаргалка (выдаётся студентам)

*Эта таблица не содержит подсказок, какие команды применять к каким задачам — только справочная информация.*

| Что делает команда | Команда |
|-------------------|---------|
| Показать IP-адреса интерфейсов | `ip addr show` |
| Показать ARP-таблицу | `ip neigh` |
| Показать таблицу маршрутизации | `ip route` |
| Показать слушающие порты (TCP) | `ss -tln` |
| Показать слушающие порты с процессами | `ss -tlpn` |
| Показать все TCP-соединения | `ss -tn` |
| Показать все UDP-порты с процессами | `ss -ulpn` |
| Показать DNS-серверы | `cat /etc/resolv.conf` |
| Показать, какой процесс открыл файл | `lsof /путь/к/файлу` |
| Показать информацию о процессе по PID | `ps -p <PID> -o pid,ppid,cmd` |
| Показать рабочую директорию процесса | `pwdx <PID>` |
| Показать путь к исполняемому файлу | `ls -l /proc/<PID>/exe` |
| Найти процессы по строке в команде | `ps aux \| grep <строка>` |
| Трассировка маршрута | `tracepath 8.8.8.8` |
| Пинг | `ping -c 4 1.1.1.1` |
| Проверить TCP-порт (bash) | `timeout 1 bash -c "echo > /dev/tcp/<host>/<port>" && echo открыт` |
