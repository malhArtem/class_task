## Задание: Создание простого REST API на FastAPI

Вам необходимо разработать простое веб-приложение на FastAPI, которое реализует REST API для управления списком книг. Приложение должно хранить данные в оперативной памяти (например, в виде словаря или списка). Все эндпоинты должны возвращать соответствующие HTTP‑статусы и корректно обрабатывать ошибки (например, попытку получить несуществующую книгу).

# Требования к функциональности


1. ## Хранилище
      Реализуйте хранение книг в памяти, например, в виде списка словарей или списка строк(названия книг). 
2. ## Эндпоинты
      Реализуйте следующие эндпоинты:
   * GET /books – возвращает список всех книг.
   * GET /books/{book_id} – возвращает информацию о книге с указанным id. Если книга не найдена – возвращать статус 404 с соответствующим сообщением.
   * POST /books – создаёт новую книгу. Принимает название книги. Возвращает созданную книгу с присвоенным id и статус 201.
   * PUT /books/{book_id} – обновляет название книги. Если книга существует – заменяет её и возвращает обновлённый объект; если нет – возвращает 404.
   * DELETE /books/{book_id} – удаляет книгу. Если книга найдена – удаляет и возвращает статус 204 (без содержимого); если нет – возвращает 404.
3. ## Документация
      Убедитесь, что при запуске приложения интерактивная документация (Swagger UI) доступна по адресу /docs.
