## Лабораторная работа: Настройка сети в операционной системе Windows (актуально для Windows 10)

**Дисциплина:** Конфигурирование Windows  
**Среда выполнения:** Виртуальная машина с Windows 7 (допускаются отличия в интерфейсе, описанные в примечаниях)  
**Цель работы:** Изучить основные сетевые настройки Windows 10, научиться настраивать статический и динамический IP-адрес, DNS, проверять сетевое соединение, управлять брандмауэром и общим доступом к ресурсам.  

### Теоретические сведения (для Windows 10)

В Windows 10 управление сетью осуществляется через **Параметры → Сеть и Интернет** (для Wi-Fi и Ethernet) или классическую **Панель управления → Центр управления сетями и общим доступом**. Для автоматического получения параметров используется протокол DHCP, для ручной настройки – статические адреса. Основные сетевые команды: `ipconfig`, `ping`, `tracert`, `nslookup`, `netstat`, `netsh`.

### Оборудование и ПО

- Виртуальная машина (VirtualBox, VMware и т.п.) с установленной Windows 7.
- Хост-ОС с доступом в интернет (для проверки связи).
- Виртуальный сетевой адаптер, настроенный на режим NAT или мост.

### Ход работы

#### Задание 1. Определение текущих сетевых параметров

1. Запустите виртуальную машину, войдите в систему.  
2. Откройте командную строку (Win + R → `cmd` → Enter).  
3. Выполните команду `ipconfig /all`. Запишите:  
   - Имя адаптера  
   - IPv4-адрес  
   - Маску подсети  
   - Основной шлюз (Default Gateway)  
   - Адреса DNS-серверов  
4. *Примечание для Windows 7:* интерфейс команды `ipconfig` идентичен.  

#### Задание 2. Настройка статического IP-адреса (аналог Windows 10)

> В Windows 10 путь: **Параметры → Сеть и Интернет → Ethernet (или Wi-Fi) → Изменение параметров адаптера** → ПКМ по адаптеру → **Свойства** → **IP версии 4 (TCP/IPv4)** → **Свойства**.

На виртуальной машине с Windows 7 выполните:

1. Откройте **Панель управления → Центр управления сетями и общим доступом → Изменение параметров адаптера**.  
2. Правой кнопкой мыши по активному адаптеру → **Свойства**.  
3. Выберите **Протокол Интернета версии 4 (TCP/IPv4)** → нажмите **Свойства**.  
4. Установите переключатель **Использовать следующий IP-адрес**. Введите:  
   - IP-адрес: `192.168.1.100`  
   - Маска подсети: `255.255.255.0`  
   - Основной шлюз: `192.168.1.1`  
5. DNS-серверы: предпочтительный `8.8.8.8`, альтернативный `8.8.4.4`.  
6. Нажмите **ОК** дважды.  
7. Проверьте связь со шлюзом: `ping 192.168.1.1`. При отсутствии ответа проверьте настройки виртуального сетевого адаптера (должен быть в режиме NAT или моста).  
8. Верните автоматическое получение IP после выполнения (переключатель **Получить IP-адрес автоматически**).

#### Задание 3. Настройка альтернативного DNS (Windows 10)

В Windows 10 можно задать резервный DNS через **Параметры → Сеть и Интернет → Дополнительные параметры сети → Дополнительные параметры** (или через старую панель).  

На Windows 7 выполните аналогично заданию 2, но оставьте IP автоматическим, а DNS укажите вручную:  
- В свойствах TCP/IPv4 выберите **Получить IP-адрес автоматически** → **Использовать следующие адреса DNS-серверов**:  
  - Предпочтительный: `1.1.1.1`  
  - Альтернативный: `9.9.9.9`  

Примените настройки. Выполните `ipconfig /all`, убедитесь, что DNS-серверы изменились.

#### Задание 4. Проверка сетевого подключения и маршрутизации

Используйте командную строку для диагностики (команды одинаковы для Win7/Win10):

1. `ipconfig` – просмотр текущих адресов.  
2. `ping ya.ru` – проверка доступа в интернет. Проанализируйте время и потери пакетов.  
3. `tracert ya.ru` – трассировка маршрута. Запишите количество хопов.  
4. `nslookup google.com` – определение IP по доменному имени (работа с DNS).  
5. `netstat -an` – просмотр активных сетевых соединений и слушающих портов.

#### Задание 5. Управление брандмауэром (Защитник Windows)

Актуально для Windows 10: через **Параметры → Обновление и безопасность → Безопасность Windows → Брандмауэр и защита сети**.  

На Windows 7:

1. Откройте **Панель управления → Брандмауэр Windows**.  
2. Нажмите **Разрешить программу или компонент через брандмауэр Windows**.  
3. Найдите в списке **Удаленный рабочий стол** и разрешите его для частной сети.  
4. Проверьте: из командной строки выполните `telnet <IP_вашей_ВМ> 3389` (если telnet не установлен, используйте `Test-NetConnection` в PowerShell, но в Win7 PowerShell может быть старым – лучше добавить правило для входящего ICMP).  
   - Разрешите **протокол ICMPv4** (ответ на ping): в свойствах брандмауэра Windows 7 → **Дополнительные параметры** → **Правила для входящих подключений** → **Общий доступ к файлам и принтерам (эхо-запрос – ICMPv4-In)** → включить.  
5. Убедитесь, что хост-машина может пропинговать ВМ.

#### Задание 6. Настройка общего доступа к папкам (сетевое обнаружение)

В Windows 10 для общего доступа используется **Сетевой общий доступ** в Параметрах.  

На Windows 7:

1. **Центр управления сетями и общим доступом → Изменение дополнительных параметров общего доступа**.  
2. Включите **сетевое обнаружение** и **общий доступ к файлам и принтерам** для частной сети.  
3. Создайте папку `C:\LabShare`, правой кнопкой → **Свойства → Доступ → Общий доступ...** → добавьте пользователя **Все** с правами **Чтение**.  
4. На другой виртуальной или хостовой машине (с той же сетью) откройте **Сеть** – должна отображаться ваша ВМ и её общая папка.  

*Примечание:* в Windows 10 по умолчанию отключено сетевое обнаружение для общедоступных сетей, обязательно выберите профиль **Частная сеть**.

#### Задание 7. Сброс стека TCP/IP (актуально для устранения неполадок)

В Windows 10 команды выполняются в командной строке от имени администратора:

```cmd
netsh int ip reset
netsh winsock reset
ipconfig /flushdns
```

После выполнения требуется перезагрузка. На Windows 7 команды работают аналогично.

Выполните их, если в процессе работы были сбои сетевого подключения.
