
### **Задание 1: Система кэширования веб-сервера**

**Требования:**
1.  При повторном запросе страницы, которая уже есть в кэше, ее нужно переместить в "самое свежее" положение.
2.  При добавлении новой страницы, если кэш полон, нужно удалить "самую старую" страницу.
3.  Быстро проверять, есть ли страница в кэше.

**Вопрос:** Какую структуру(-ы) данных вы выберете? Обоснуйте выбор и опишите, как будут работать операции добавления, получения и удаления.

**Требования:**

2.  Быстро находить запись по имени.
3.  **Регулярно (например, раз в секунду) удалять просроченные записи (у которых `текущее_время - время_добавления > TTL`)**.
4.  TTL может быть разным для разных записей.
