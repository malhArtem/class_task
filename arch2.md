

# Лабораторная работа: «Проектирование микросервисной архитектуры с бюджетом» (40 минут)

## Часть 1. Теория: При каком RPS нужен каждый компонент

### Таблица 1. Пороги нагрузки для компонентов

| Русское название | Назначение | При каком RPS нужен |
|------------------|------------|---------------------|
| **Балансировщик нагрузки** | Распределяет запросы между несколькими экземплярами одного сервиса. | **>100 RPS** на любой сервис. Один экземпляр FastAPI вывозит ~1000 RPS, но балансировщик нужен, когда экземпляров становится 2+ (т.е. при >1000 RPS на сервис). |
| **Шлюз API** | Единая точка входа для клиентов. Централизует аутентификацию, лимиты, логирование. | **>500 RPS** суммарно по системе **ИЛИ** >3 микросервисов **ИЛИ** >1 команды разработчиков. |
| **Реестр сервисов** | Автоматически находит адреса сервисов, которые перезапускаются и меняют IP. | **>1000 RPS** **ИЛИ** >10 экземпляров сервисов **ИЛИ** автодемасштабирование. |
| **Брокер сообщений** | Асинхронная связь между сервисами. | **>100 RPS** на пару «отправитель-получатель» **ИЛИ** когда получатель медленнее отправителя **ИЛИ** нужна гарантия доставки. |
| **Предохранитель** | Защита от зависаний при вызове удалённых сервисов. | **>50 RPS** на вызовы к внешнему сервису **ИЛИ** >2 сервисов в цепочке. |
| **Распределённая трассировка** | Отслеживает путь запроса через сервисы. | **>500 RPS** **ИЛИ** >4 сервисов в цепочке. |
| **Централизованное логирование** | Сбор логов со всех сервисов в одно место. | **Любая нагрузка, но >3 сервисов** **ИЛИ** >2 разработчиков. |
| **Центр конфигурации** | Центральное хранение настроек. | **>10 сервисов** **ИЛИ** >3 окружений (dev/stage/prod). |
| **Сбор метрик + оповещения** | Мониторинг здоровья системы. | **Любая продакшн-нагрузка (>0 RPS)**. |

### Таблица 2. Мощность одного экземпляра FastAPI

| Параметр | Значение |
|----------|----------|
| RPS на 1 ядро (лёгкие запросы) | ~1000 |
| RPS на 1 ядро (средние запросы с БД) | ~300-500 |
| RPS на 1 ядро (тяжёлые запросы, внешние вызовы) | ~100-200 |

> Для простоты в этой лабораторной считаем, что все запросы лёгкие, и **1 экземпляр FastAPI = 1000 RPS**.

### Что такое балансировщик и чем он отличается от шлюза API

| | Балансировщик | Шлюз API |
|--|---------------|----------|
| **Что делает** | Распределяет трафик между экземплярами **одного** сервиса | Маршрутизирует запросы между **разными** сервисами |
| **Знает про бизнес-логику?** | Нет | Да (токены, заголовки, склейка ответов) |
| **Когда нужен** | Всегда, когда >1 экземпляра сервиса | Когда >1 сервиса и есть внешние клиенты |

**Правильная связка:** Балансировщик → Шлюз API → Микросервисы
