## Модуль 4. Разработка информационной системы (Учёт оборудования)

### Цель

Разработать приложение для учёта компьютерного и офисного оборудования: авторизация, две роли («администратор» и «пользователь»), возможность просматривать, добавлять и изменять инвентарные единицы.

### 4.1. База данных

#### База данных состоит из двух таблиц.

#### Таблица «Пользователи» хранит учётные записи сотрудников.

* Идентификатор — уникальный номер пользователя.
* Логин — уникальное имя для входа.
* Пароль — хранится в открытом виде (для упрощения экзаменационного задания).
* Полное имя — ФИО сотрудника.
* Роль — может быть либо «администратор», либо «пользователь».

#### Таблица «Оборудование» содержит данные об инвентарных единицах.

* Идентификатор — уникальный номер записи.
* Наименование — тип оборудования (например, «Ноутбук», «Монитор»).
* Модель — конкретная модель устройства.
* Инвентарный номер — уникальный номер, присвоенный организации.
* Статус — состояние оборудования: «в работе», «списано»

### 4.2. Функционал

#### Общее:

* Форма входа: логин/пароль, проверка.
* При успешном входе открывается главное окно.

#### Администратор:

* Может добавлять, редактировать и удалять оборудование.
* Может просматривать таблицу оборудования
* Может изменять статус оборудования.

#### Пользователь:

* Видит все оборудование
* Не может добавлять или удалять оборудование

### Интерфейс:

* После входа – таблица оборудования.
* Для администратора – кнопки «Добавить», «Редактировать», «Удалить».
